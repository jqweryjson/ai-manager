# Анализ стратегии авторизации

## Текущая ситуация

### Проблемы текущей реализации

1. **Два разных пользователя для одного человека:**
   - Google OAuth (веб) → пользователь с `email` + `google_id`
   - Telegram Mini App (мобильный) → пользователь с `telegram_id` + временный email
   - Результат: один человек = два аккаунта

2. **Бесшовная авторизация через Telegram Mini App:**
   - ✅ Работает для новых пользователей (создается аккаунт автоматически)
   - ❌ Не работает для существующих пользователей (создается новый аккаунт)
   - ❌ Нет связи между веб и мобильным

3. **Вход через email/password:**
   - Сейчас не реализован (TODO в коде)
   - Не используется

## Анализ вариантов

### Вариант 1: Telegram Login Widget для веб (рекомендуется)

**Идея:** Использовать официальный Telegram Login Widget для веб-версии.

**Как работает:**

1. На веб-версии: пользователь нажимает "Войти через Telegram"
2. Открывается официальный виджет Telegram (iframe или popup)
3. Пользователь авторизуется через Telegram
4. Получаем `telegram_id` и создаем/находим пользователя
5. На мобильном: при входе в Mini App используем тот же `telegram_id` из `initData`
6. Результат: **один пользователь, один аккаунт** ✅

**Преимущества:**

- ✅ Один провайдер (Telegram) = один идентификатор
- ✅ Нет проблемы связывания аккаунтов
- ✅ Бесшовная авторизация между веб и мобильным
- ✅ Не нужен SMS-провайдер
- ✅ Простая реализация

**Минусы:**

- Пользователи без Telegram не смогут войти (но это может быть приемлемо)
- Нужно убрать Google OAuth (или оставить как опцию)

**Реализация:**

```typescript
// На фронтенде: добавить Telegram Login Widget
<script async src="https://telegram.org/js/telegram-widget.js?22"
  data-telegram-login="your_bot_username"
  data-size="large"
  data-onauth="onTelegramAuth(user)"
  data-request-access="write">
</script>

// Обработчик
function onTelegramAuth(user) {
  // Отправляем user на backend
  fetch('/api/auth/telegram-web', {
    method: 'POST',
    body: JSON.stringify(user)
  });
}

// На backend: валидация и авторизация
fastify.post("/auth/telegram-web", async (request, reply) => {
  const { id, first_name, last_name, username, photo_url, auth_date, hash } = request.body;

  // Валидация подписи (как в Mini App)
  if (!validateTelegramWebAuth({ id, first_name, auth_date, hash })) {
    return reply.status(401).send({ error: "Invalid signature" });
  }

  // Ищем/создаем пользователя по telegram_id
  let user = await findUserByTelegramId(id.toString());
  if (!user) {
    user = await createUser({
      email: `tg_${id}@telegram.local`,
      name: `${first_name} ${last_name || ''}`,
      telegramId: id.toString(),
    });
  }

  // Генерируем токены
  // ...
});
```

### Вариант 2: Упрощенное связывание (без SMS)

**Идея:** Оставить Google OAuth для веб, но упростить связывание аккаунтов.

**Как работает:**

1. На веб: пользователь авторизуется через Google OAuth
2. На мобильном: пользователь входит через Telegram Mini App
3. При входе в Mini App проверяем номер телефона в `telegram_accounts`
4. Если найден другой пользователь с таким номером → показываем диалог
5. Пользователь подтверждает → связываем аккаунты (без SMS)

**Преимущества:**

- ✅ Не нужно убирать Google OAuth
- ✅ Не нужен SMS-провайдер
- ✅ Пользователь контролирует процесс

**Минусы:**

- Требует дополнительных действий от пользователя
- Нужно реализовать UI для связывания

### Вариант 3: Гибридный подход

**Идея:** Комбинация обоих вариантов.

**Сценарии:**

1. **Новый пользователь:**
   - Может выбрать: Telegram Login Widget или Google OAuth
   - Рекомендуем Telegram (проще, бесшовная авторизация)

2. **Существующий пользователь (Google OAuth):**
   - Может продолжать использовать Google
   - При входе в Mini App предлагаем связать аккаунты

3. **Пользователь настроил Telegram интеграцию:**
   - После подключения Telegram аккаунта проверяем номер телефона
   - Если найден другой пользователь → предлагаем связать

## Рекомендация

### Краткосрочное решение (простое)

**Использовать Telegram Login Widget для веб:**

1. ✅ Убрать форму входа через email/password (она и так не работает)
2. ✅ Добавить Telegram Login Widget на страницу авторизации
3. ✅ Оставить Google OAuth как опцию (для существующих пользователей)
4. ✅ При авторизации через Telegram (веб или Mini App) использовать `telegram_id`
5. ✅ Результат: новые пользователи через Telegram = один аккаунт на всех устройствах

**Преимущества:**

- Простая реализация
- Нет проблемы связывания для новых пользователей
- Бесшовная авторизация между веб и мобильным

### Долгосрочное решение (полное)

**Добавить упрощенное связывание для существующих пользователей:**

1. Реализовать Telegram Login Widget (как выше)
2. Добавить логику связывания через UI (без SMS):
   - При входе в Mini App проверять номер телефона
   - Показывать диалог: "Обнаружен аккаунт с email: user@example.com. Связать?"
   - Пользователь подтверждает → автоматически связываем
3. При настройке Telegram интеграции:
   - После успешного подключения проверять номер телефона
   - Если найден другой пользователь → показывать диалог в UI

## Ответы на вопросы

### 1. Правильно ли сделана бесшовная авторизация через Telegram Mini App?

**Частично правильно:**

- ✅ Работает для новых пользователей
- ❌ Не работает для существующих (создает дубликат)
- ❌ Нет связи между веб и мобильным

**Решение:** Использовать Telegram Login Widget для веб → один `telegram_id` на всех устройствах.

### 2. Отказаться от входа через почту?

**Да, рекомендуется:**

- Форма входа через email/password не реализована (TODO)
- Не используется
- Можно убрать, оставить только провайдеры (Telegram, Google)

### 3. Проверка активной сессии между веб и мобильным?

**Проблема:** Telegram Mini App не может проверить активную сессию в веб-версии напрямую.

**Решение:** Использовать `telegram_id` как общий идентификатор:

- На веб: Telegram Login Widget → получаем `telegram_id`
- На мобильном: Telegram Mini App → получаем `telegram_id` из `initData`
- Оба используют один и тот же `telegram_id` → автоматически один аккаунт ✅

**Альтернатива:** Если пользователь авторизован через Google на веб, а потом заходит через Telegram Mini App:

- Проверяем номер телефона в `telegram_accounts`
- Если найден другой пользователь → показываем диалог связывания
- Пользователь подтверждает → связываем (без SMS, просто UI диалог)
