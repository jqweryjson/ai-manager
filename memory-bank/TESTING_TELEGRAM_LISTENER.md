# Тестирование Telegram Listener

## Что нужно для тестирования

1. ✅ **В БД должен быть аккаунт со статусом `connected`**
2. ✅ **Должна быть хотя бы одна подписка с `enabled = true`**
3. ✅ **TypeScript должен быть скомпилирован** (`npm run build`)

## Вариант 1: Локальный запуск (рекомендуется для первого теста)

### Шаг 1: Проверь переменные окружения

```bash
# Убедись, что есть .env файл с:
DATABASE_URL=postgresql://postgres:postgres123@localhost:8005/ai_manager
JWT_SECRET=твой_секрет
BACKEND_SECRET=твой_секрет_для_шифрования
```

### Шаг 2: Запусти listener

```bash
npm run dev:listener
```

**Что должно произойти:**

```

📋 Найдено 1 аккаунтов для прослушивания

✅ Инициализировано 1 слушателей Telegram
```

## Проверка данных в БД

Если listener не находит аккаунты, проверь:

```sql
-- Проверь, что есть connected аккаунт
SELECT id, user_id, status, phone
FROM telegram_accounts
WHERE status = 'connected';

-- Проверь, что есть enabled подписка
SELECT ts.*, ta.phone
FROM telegram_subscriptions ts
INNER JOIN telegram_accounts ta ON ta.id = ts.telegram_account_id
WHERE ts.enabled = true
  AND ta.status = 'connected';
```

## Troubleshooting

### Проблема: "Найдено 0 аккаунтов"

**Решение:**

- Проверь, что в БД есть аккаунт со статусом `connected`
- Проверь, что есть подписка с `enabled = true` для этого аккаунта

### Проблема: "Не удалось запустить слушатель"

**Решение:**

- Проверь, что `BACKEND_SECRET` установлен (нужен для расшифровки сессии)
- Проверь логи на ошибки расшифровки
- Проверь, что PostgreSQL доступен

### Проблема: Сообщения не появляются в консоли

**Решение:**

- Убедись, что отправляешь сообщение в **подключенный чат** (где `enabled = true`)
- Проверь, что `workspace_id` и `role_id` установлены в подписке
- Проверь логи на ошибки обработки сообщений

### Проблема: TIMEOUT ошибки

**Решение:**

- Это нормально, GramJS пытается переподключиться автоматически
- Проверь интернет-соединение
- Убедись, что Telegram не заблокирован
