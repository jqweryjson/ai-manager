Что это за система
Это умный поиск по знаниям компании. Пользователь задаёт вопрос своими словами — система находит подходящие фрагменты из внутренних материалов и собирает ответ строго на их основе, всегда с цитатами и ссылками на источник.

Как появляется ответ (простыми шагами)

1. Принимаем вопрос и при необходимости уточняем формулировку.
2. Ищем подходящие знания в базе (векторы в Milvus).
3. Отбираем самые точные фрагменты.
4. Собираем связный ответ на их основе (модель Groq), без «додумываний».
5. Прикладываем цитаты/ссылки на источник.
6. Если данных мало — спрашиваем уточнение вместо догадок.

Из чего состоит MVP

- Веб‑сервис (Fastify.js) — принимает вопросы и отдаёт ответы, поддерживает стриминг.
- Поиск знаний (Milvus) — быстро находит похожие по смыслу фрагменты.
- Сборка ответа (Groq API) — формирует понятный ответ строго по найденному.
- Кэш и сессии (Redis) — ускоряет повторные обращения, хранит контекст диалога.
- Хранилище текстов — исходные материалы компании (txt/markdown на старте).

Пользовательская изоляция

- Каждый загруженный объект и каждый чанк принадлежат конкретному `user_id`.
- Ответы строятся только из данных текущего пользователя.
- Общего доступа/шаринга нет.

Как добавлять знания
Через API `/ingest`: загружаем текстовые файлы, система автоматически делит их на смысловые части, очищает, удаляет дубликаты, создаёт «отпечатки‑векторы» и добавляет в поиск.

Как контролируем качество

- Всегда показываем источники (цитаты).
- Меряем «покрытие контекстом» — долю ответа, подтверждённую цитатами.
- Если покрытие низкое — просим уточнить вопрос, а не придумываем ответ.

Как растём без переделок

- Поддержка новых форматов (PDF, CRM) — через коннекторы к тому же конвейеру загрузки.
- Точность можно повысить ранжировщиком (reranker) и гибридным поиском.
- Масштабирование: каждый блок можно запустить в нескольких экземплярах.

Ограничения MVP (честно)

- На старте только текст (txt/markdown) и один профиль эмбеддингов.
- Нет сложных ролей и разграничения доступа — добавим позже.
